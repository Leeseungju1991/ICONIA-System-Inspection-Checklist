# System Inspection Checklist

> 프로젝트 운영 안정성, 보안성, 확장성 및 품질 보증을 위한 종합 점검 문서

## 📌 검토 범위

- HW / FW
- Server
- AI
- App
- Admin

---

# 1. 아키텍처 및 설계 검증

## 1.1 시스템 구조

- [ ] 모듈 분리 적절성 검토
- [ ] 의존성 최소화 여부 확인
- [ ] 확장 가능한 구조 설계 여부
- [ ] 단일 장애점(SPOF) 존재 여부 점검
- [ ] MSA / Monolithic 구조 적합성 검토

## 1.2 코드 품질

- [ ] 코드 컨벤션 통일 여부
- [ ] 중복 코드 제거 여부
- [ ] 함수 / 클래스 책임 분리 여부
- [ ] 하드코딩 제거 여부
- [ ] 기술 부채 목록화 여부

## 1.3 설계 문서화

- [ ] 시스템 아키텍처 문서 최신화
- [ ] API 명세서 작성 여부
- [ ] ERD / DB 설계 문서 존재 여부
- [ ] 시퀀스 다이어그램 작성 여부
- [ ] 배포 구조도 최신화 여부

---

# 2. 보안(Security)

## 2.1 인증 / 인가

- [ ] JWT 만료 처리 구현 여부
- [ ] 세션 탈취 방어 적용 여부
- [ ] RBAC 권한 분리 여부
- [ ] 관리자 권한 제한 적용 여부
- [ ] MFA 적용 여부

## 2.2 입력값 검증

- [ ] SQL Injection 방어 여부
- [ ] XSS 방어 여부
- [ ] CSRF 방어 여부
- [ ] Path Traversal 방어 여부
- [ ] 파일 업로드 검증 여부
